# Session 2026-04-13-h — REPL Replace Mode Hardening

## What Was Done

Extended Phase 2b-gaps-3 with replace mode support and fresh session testing.

### Replace Mode Architecture (confirmed from CC source)
- Ant build: USER_TYPE=ant + cli entrypoint triggers isReplModeEnabled()
- getTools() filters REPL_ONLY_TOOLS: Read, Write, Edit, Bash, Glob, Grep, NotebookEdit, Agent
- getUsingYourToolsSection() returns minimal text (just TaskCreate)
- REPL prompt is sole guide for all file/shell operations
- Our implementation: config.json repl.mode="replace", binary-patched loader stashes then filters primitives

### Code Changes
- **repl.js glob():** Catch-all patterns (*, **/, **) now drop --glob flag — rg --files alone respects .gitignore correctly. Previously glob("**/*") returned 79K files.
- **repl.js prompt():** Mode-aware — calls coexistPrompt() or replacePrompt() based on config
- **replacePrompt():** Comprehensive prompt incorporating primitive tool guidance (Read 256KB limit, Edit uniqueness, Grep regex, Write semantics), battle-hardened codebase scan pattern (git ls-files, binary filtering, try/catch), JS-not-Python note, "Return summaries" section
- **governance.ts:** Binary patch text changed to pre-action check framing ("Before issuing multiple tool calls, ask: could one REPL call do this?")
- **Verification signatures:** Updated across governance.ts and index.ts

### Fresh Session Tests (4 attempts)
1. Coexist mode, strengthened prompts: Model reasoned about REPL but used Bash (FAIL)
2. Coexist mode, stronger language: Same pattern — acknowledges REPL, uses Bash (FAIL)
3. Replace mode, first attempt: Model used REPL exclusively but took 7 iterations (PARTIAL)
4. Replace mode, hardened prompt: Single REPL call, comprehensive dashboard, 1m42s (PASS)

### Key Discoveries
- rg --glob "**/*" bypasses .gitignore — the catch-all pattern acts as an include-override
- CC's native GlobTool ALSO uses --no-ignore --hidden by default (glob.ts lines 98-99)
- CC Read tool: 256KB maxSizeBytes (throws pre-read), 25K maxTokens (throws post-read), override via CLAUDE_CODE_FILE_READ_MAX_OUTPUT_TOKENS env var
- JSONL transcripts don't include tool_use blocks — only thinking and text blocks persisted
- Coexist mode's "prefer REPL" prompt guidance cannot override model's trained Bash instinct — the model plans REPL but executes Bash every time
