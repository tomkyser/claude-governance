## Key Findings

### Prompt Override Root Cause
The entire override pipeline was working correctly — pieces matching, regex generation, replacement — but the REPLACEMENT CONTENT was stock text. The override markdown files never made it from the repo to the config dir. A deployment gap, not a matching gap.

### Zod Passthrough Simplicity
The CC pipeline only calls `.safeParse()` and `.parse()` on `inputSchema`. A 2-line object shim is sufficient — no need to find or construct a real Zod schema. The MCPTool passthrough approach (finding `z.object({}).passthrough()` in the binary) would have been fragile and unnecessary.

### Embedded Tools Exclusion Pattern
Only ONE match in the entire 12.8M character binary for the conditional exclusion ternary `fn()?[]:[X,Y]`. This makes it a reliable verification signature. The pattern is inside the preserved `_b` array — our concat never re-adds Glob/Grep.
